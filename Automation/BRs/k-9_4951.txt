Comment#6:
I looked into this some more. The "Disney+ Email" text is showing because the `<table>` tag starts the body and everything that is meant to be part of `<head>` ends up in the body, including the `<title>` element. Because `<title>` is not on our list of allowed tags for the body, it is filtered out and only the text remains.
Interestingly browsers' default stylesheet contains a rule to hide the contents of title elements. See [this blog post](https://mathiasbynens.be/notes/css-hidden-elements). If we don't filter out `<title>` tags the text will be hidden. I see no harm in making that change.

The second issue is that we discard all attributes from the `<body>` element. That's not good and something we will fix as well.

But even with these changes there will remain a tiny gap at the top for now. That's because of the layout of the message view screen. We'll get to overhauling that eventually.
