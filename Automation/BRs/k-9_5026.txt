Bug Report Title:
Unable to delete messages from inbox

Bug Report Issue:
**Describe the bug**
When trying to delete messages, messages disappear, but are not moved into Trash directory.
When re-opening k-9, deleted messages still appear in inbox.

Please note that it could also happen with messages from other directory ( I didn't test it)

**To Reproduce**
Steps to reproduce the behavior:
1. Open k-9 (unified inbox in my case)
2. Select one or many messages
3. Delete them
4. Quit k-9
5. Re-open k-9
6. Messages are still in inbox

**Expected behavior**
Deleted messages should be visible in Trash directory and not in inbox anymore

**Environment (please complete the following information):**
 - K-9 Mail version: 5.722 (beta)
 - Android version: 11
 - Device: Google Pixel 4a
 - Account type: IMAP

**Additional context**
Add any other context about the problem here.
